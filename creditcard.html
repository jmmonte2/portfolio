Credit Card Validation

About
Java application that generates random credit card numbers that are validated by Luhn's algorithm. Each time a card is created they are stored in an SQLite database table.

Motivation
The reason I decided to work on this project was to gain an understanding of how Java applications communicate with databases, specifically, to learn how to store objects from an application into a database.

Key Takeaways
JDBC
Gained an understanding of how JDBC API is responsible for connecting Java applications to a database. In this project, I utilized the SQLite JDBC driver, which was used to convert JDBC calls to a call that the database can understand.

Closing Connection
Also learned that it is best practice to explicitly close the database connection after using it, since a memory leakage can occur if left open and that could potentially lead to poor performance in application.

SQL Statements
This project also taught me how to run SQL statements using the necessary library (java.sql.Statement) while also making use of exception handling (try-catch statements) to run the statements just in case runtime errors occur.
